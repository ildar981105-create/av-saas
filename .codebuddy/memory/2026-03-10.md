## 全自动流程改造（translate-v4.html）

### 改动概要

将视频译制的对话配置流程从"逐步确认"改为**全自动执行**：

### 三项改动

1. **去掉中间确认步骤**
   - 擦除、字幕、配音每步完成后自动确认并进入下一步（1.2s 延迟）
   - 字幕步骤：识别完成 → 自动进入翻译 → 翻译完成 → 自动确认
   - 最后一步完成后自动开始处理（不再显示"开始处理"/"去精调"按钮）
   - 去掉了所有 `chat-step-confirm`、`chat-step-skip`、`chat-step-ft-entry` 按钮的 HTML 生成
   - `confirmStep()` 中改为不显示用户气泡
   - `showStartButton()` 改为自动调用 `startProcessing()`

2. **文案整理**
   - 分析完成提示：→"即将自动完成...全流程，完成后可在精调面板中微调细节"
   - phaseLabel："AI 自动配置中..."
   - 聊天头部："上传视频自动处理"
   - 步骤 autoDesc 和 preDesc 更新为自动执行风格
   - 翻译配置气泡："语音识别完成，正在自动翻译"
   - 开始处理气泡：AI 自动触发风格

3. **修复列表布局溢出**
   - `.chat-sub-line` / `.chat-erase-line` / `.chat-voice-line`：添加 `overflow: hidden; min-width: 0`
   - `.csl-orig`：添加 `overflow: hidden; text-overflow: ellipsis; white-space: nowrap; min-width: 0`
   - 三种列表容器（`-list-inline`）：添加 `overflow: hidden`

## 修复卡片不可见 + 增加流程过渡描述

### 问题

- `.chat-bubble` 上添加的 `overflow: hidden` 导致所有卡片内容被裁切不可见
- 自动流程中缺少过渡描述，用户不知道当前完成了什么、接下来要做什么

### 修复

1. **去掉 `.chat-bubble` 的 `overflow: hidden`**——溢出控制只在列表行和容器上做
2. **`confirmStep()` 增加过渡描述气泡**——根据当前步骤和下一步生成具体文案：
   - 擦除完成 → "字幕擦除区域检测已就绪 ✓ / 接下来自动执行文案识别与翻译..."
   - 字幕完成 → "文案翻译已就绪 ✓ / 接下来自动执行 AI 配音生成..."
   - 配音完成 → "配音生成已就绪 ✓ / 全部步骤已完成，即将自动开始处理..."
3. **修正步骤 badge**：从固定 `X / 3` 改为 `X / activeSteps.length`（适配单步模式）
4. **翻译配置气泡文案**：改为"文案已就绪，正在以默认配置自动翻译"
